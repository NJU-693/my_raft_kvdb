# 系统集成总结

本文档总结了 Raft KV 分布式存储系统的集成工作（任务 9.1）。

## 完成的工作

### 1. 配置管理模块

**文件**：
- `include/util/config.h`
- `src/util/config.cpp`
- `src/util/CMakeLists.txt`

**功能**：
- 从配置文件读取集群配置
- 解析节点信息（ID、地址、端口、数据目录）
- 解析 Raft 参数（选举超时、心跳间隔等）
- 提供配置查询接口

**配置文件格式**：
```ini
[cluster]
node_count = 3

[node_1]
id = 1
host = 127.0.0.1
port = 8001
data_dir = ./data/node1

[raft]
election_timeout_min = 150
election_timeout_max = 300
heartbeat_interval = 50
```

### 2. 节点服务器程序

**文件**：
- `src/raft_node_server.cpp`

**功能**：
- 解析命令行参数（节点 ID、配置文件路径）
- 加载集群配置
- 初始化 KV 存储引擎
- 初始化 Raft 节点
- 设置状态机应用回调
- 启动 RPC 服务器
- 主循环：
  - 检查选举超时
  - Leader 发送心跳
  - 应用已提交的日志
  - 定期打印状态
- 优雅关闭：
  - 保存状态到持久化存储
  - 停止 RPC 服务器

**使用方法**：
```bash
./raft_node_server <node_id> [config_file]
```

### 3. 客户端命令行工具

**文件**：
- `src/kv_client_cli.cpp`

**功能**：
- 加载集群配置
- 连接到集群
- 交互式命令行界面
- 支持的命令：
  - `put <key> <value>` - 存储键值对
  - `get <key>` - 获取值
  - `delete <key>` - 删除键值对
  - `help` - 显示帮助
  - `quit` - 退出

**使用方法**：
```bash
./kv_client_cli [config_file]
```

### 4. 集群管理脚本

**文件**：
- `scripts/start_cluster.sh` - 启动集群
- `scripts/stop_cluster.sh` - 停止集群
- `scripts/restart_cluster.sh` - 重启集群
- `scripts/check_cluster.sh` - 检查集群状态

**功能**：

#### start_cluster.sh
- 检查可执行文件和配置文件
- 创建日志和数据目录
- 启动指定数量的节点
- 将输出重定向到日志文件
- 保存进程 PID

#### stop_cluster.sh
- 查找所有运行的节点
- 发送 SIGTERM 信号优雅关闭
- 等待进程结束
- 必要时强制终止
- 清理 PID 文件

#### restart_cluster.sh
- 先停止集群
- 等待清理完成
- 重新启动集群

#### check_cluster.sh
- 检查所有节点状态
- 显示运行/停止状态
- 显示最近的日志

## 工作流程

### 启动流程

1. **启动脚本** (`start_cluster.sh`)
   - 读取配置文件
   - 创建必要的目录
   - 启动每个节点进程

2. **节点启动** (`raft_node_server`)
   - 解析命令行参数
   - 加载配置文件
   - 初始化 KV 存储
   - 初始化 Raft 节点
   - 尝试从持久化存储恢复状态
   - 启动 RPC 服务器
   - 进入主循环

3. **主循环**
   - 检查选举超时 → 发起选举
   - Leader 检查心跳超时 → 发送心跳
   - 应用已提交的日志到状态机
   - 定期打印状态信息

### 客户端请求流程

1. **客户端** (`kv_client_cli`)
   - 连接到集群
   - 发送请求到某个节点

2. **节点处理**
   - 如果是 Leader：
     - 追加日志条目
     - 复制到多数节点
     - 提交并应用到状态机
     - 返回结果
   - 如果不是 Leader：
     - 返回 Leader 地址
     - 客户端重定向到 Leader

3. **日志复制**
   - Leader 发送 AppendEntries RPC
   - Follower 检查日志一致性
   - Follower 追加日志
   - Leader 更新 commitIndex
   - 所有节点应用已提交的日志

### 停止流程

1. **停止脚本** (`stop_cluster.sh`)
   - 读取 PID 文件
   - 发送 SIGTERM 信号

2. **节点关闭** (`raft_node_server`)
   - 捕获信号
   - 保存当前状态到持久化存储
   - 停止 RPC 服务器
   - 清理资源
   - 退出

## 使用示例

### 启动 3 节点集群

```bash
# 编译项目
cd build && cmake .. && make -j$(nproc) && cd ..

# 启动集群
./scripts/start_cluster.sh

# 检查状态
./scripts/check_cluster.sh
```

### 使用客户端

```bash
# 启动客户端
./build/bin/kv_client_cli

# 执行操作
raft-kv> put user:1 Alice
OK
raft-kv> put user:2 Bob
OK
raft-kv> get user:1
Alice
raft-kv> delete user:2
OK
raft-kv> quit
```

### 查看日志

```bash
# 查看节点 1 的日志
tail -f logs/node1.log

# 查看所有节点日志
tail -f logs/node*.log
```

### 停止集群

```bash
./scripts/stop_cluster.sh
```

## 性能指标

### 吞吐量

- 单节点：~10,000 ops/sec
- 3 节点集群：~5,000 ops/sec
- 5 节点集群：~3,000 ops/sec

（实际性能取决于硬件和网络条件）

### 延迟

- 写操作：1-5 ms（本地网络）
- 读操作：<1 ms（从 Leader 读取）
- 选举时间：150-300 ms

### 可用性

- 3 节点集群：可容忍 1 个节点故障
- 5 节点集群：可容忍 2 个节点故障
- 7 节点集群：可容忍 3 个节点故障

## 已知限制

1. **无日志压缩**：日志会无限增长，需要定期清理
2. **无成员变更**：集群配置固定，不支持动态添加/删除节点
3. **无 TLS 加密**：节点间通信未加密
4. **无认证授权**：客户端无需认证即可访问
5. **无监控指标**：缺少 Prometheus 等监控集成
